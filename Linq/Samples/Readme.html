Samples  

These samples are an application that provides simple examples of the product using.

They demonstrate how applications built by help of the LINQ technology work and show good design and implementation practices that you can employ with our product. By examining the source code of these samples you can see how our advanced features help you to build a centralized data access and storage solution. Particularly, it illustrates common issues like LINQ-queries execution, Master/Detail relationships, SubmitChanges, and so on. Each of issues is presented on the single tab.

LinqConnect demos are based on the CRM_Demo database. Before launching demos first time, execute the crm_demo_<your_database_name>.sql file, supplied with the demos, to create a database. To see how demo works, launch a demo solution from the Start menu, build it, and start it.

Sample Description
Queries This sample shows you how to execute LINQ-queries and bind grid by this data. The source code displays simple LINQ-queries and method of data binding. The LINQ statement can be viewed at the bottom of page.
Master-Detail relationship Master-Detail is a model which provides data from two or more related tables. First table contains general information, other tables contain information related to the main table. So, if we change selection in main table, related datasets changed too.
Submit changes The SubmitChanges computes the set of modified objects to be inserted, updated, or deleted, and executes the appropriate commands to implement the changes to the database. You can change information about company on this page. Change value of the grid cell and confirm your changes by pressing the ENTER key. To save your changes click the Update button on the top toolbar. The event handler of this button contains the SubmitChanges method call.
